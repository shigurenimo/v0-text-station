# Teammate とのやりとり

各 teammate とどのようにやりとりするかを定義する。

## 用語の定義

- **あなた**: このスキル（copilot）を実行している Claude。技術的な判断を行い、teammate を管理する
- **ユーザー**: 対話の相手（人間）。ビジネス判断を行い、承認や重要な変更の判断を行う
- **teammate**: あなたが起動するサブエージェント（task-planner, code-scan, docs-scan, issue-{番号}）

## 動作モードの切り替え

copilot には3つのモードがある。状況に応じて即座に切り替える。

### 自動モード（自走）

タスクが以下の条件を満たす場合、ユーザーに確認せず自動で進める:

- 機能開発が発生しない（バグ修正・リファクタリング等）
- 実装方針が明確
- ビジネス判断が不要
- 仕様変更が不要

自動モードでは: Notion ステータスを「作業中_CLAUDE」に更新 → 実装を teammate に委任 → 完了後に PR 作成 → ステータスを「作業確認待ち」に更新 → `copilot-notification` で通知 → ユーザーに報告

### 中止モード

以下のいずれかに該当した時点で即座に中止する:

- 機能開発が発生する（新機能・既存機能の拡張）
- ビジネス判断が必要
- 仕様変更が必要
- 人間の承認が必要

中止時: Notion ステータスを「作業中止」に更新 → 承認が必要な場合は承認タスクをサブタスクとして作成（`notion-operations.md` 参照） → `copilot-notification` で通知 → ユーザーに理由を報告 → 次のアクションを提案

### 対話モード

ユーザーと一緒に作業するとき（引数なし起動、または中止後の継続）は `AskUserQuestion` で確認しながら進める。

## 常駐 teammate

### task-planner

新機能の仕様計画を担当。計画を提案したら終了する。

**起動方法:**

Task ツールで起動する。prompt には要望の内容と「`copilot-task-planner` スキルを展開して作業を開始してください」という指示を含める。

**処理フロー:**

1. 仕様計画を作成（機能の目的、背景、要件を定義）
2. 仕様計画を成果物として提出して終了

**あなたの対応:**

1. 仕様計画を受け取る
2. Notion タスクを作成（ステータス: 計画待ち）
3. ユーザーに提案を報告

### code-scan

既存製品の不具合を探して報告する（修正は行わない）。

**起動方法:**

Task ツールで起動する。prompt には「`copilot-code-scan` スキルを展開して作業を開始してください」という指示を含める。

**処理フロー:**

1. 問題を発見・分析
2. 成果物として提出して終了

**あなたの対応:**

1. 問題報告を受け取る
2. 自動/中止モードの判断基準で分類
3. 自動対応可能: Notion タスクを作成（作業待ち）→ GitHub Issue 作成 → issue-{番号} を起動
4. 機能開発が発生する: 中止モードに切り替えてユーザーに報告

### docs-scan

`.docs/` の仕様書を読んで問題や矛盾を発見して報告する（修正は行わない）。

**起動方法:**

Task ツールで起動する。prompt には「`copilot-docs-scan` スキルを展開して作業を開始してください」という指示を含める。

**処理フロー:**

1. 仕様の問題や矛盾を発見・分析
2. 成果物として提出して終了

**あなたの対応:**

1. 問題報告を受け取る
2. Notion タスクを作成（作業待ち）→ GitHub Issue 作成 → issue-{番号} を起動

## 動的 teammate

### issue-{番号}

Issue を実装してコミットまで完結する。GitHub 操作（PR・Issue）は行わない。

**起動方法:**

Task ツールで起動する。prompt には Issue 番号、Issue 本文、use-worktree の値、および「`copilot-issue` スキルを展開して作業を開始してください」という指示を含める。

use-worktree の判断基準:

- 自動モード（並走）: `use-worktree=true`
- 対話モードで Issue が1件: `use-worktree=false`
- 対話モードで Issue が2件以上同時: `use-worktree=true`

**処理フロー:**

1. Issue 番号と本文を受け取る
2. 実装計画を作成して報告する
3. あなたが承認したら実装を開始。拒否された場合は計画を修正して再報告
4. 以下のいずれかに該当すると判断した場合は即座に放棄してあなたに報告:
   - 機能開発が発生する
   - 人間の承認が必要
   - 仕様変更が必要
   - ビジネス判断が必要
5. 自走可能と判断した場合: （use-worktree=true なら worktree 作成 →）実装 → 動作確認 → コミット → 成果物を報告
6. クリーンアップ指示を受けたら（use-worktree=true なら worktree を削除して）終了

**あなたの対応（計画報告時）:**

- 計画を確認して承認または拒否を返す
- 中止条件に該当する場合は承認せず中止モードに切り替える

**あなたの対応（完了報告時）:**

1. 成果物（ブランチ名・コミットハッシュ・動作確認結果）を受け取る
2. PR を作成する（`gh-pull-request` スキルに従う）
3. Notion ステータスを「作業確認待ち」に更新
4. `copilot-notification` で通知
5. ユーザーに報告
6. PR マージ後: Notion ステータスを「完了」に更新 → `copilot-notification` で通知 → クリーンアップを指示

**あなたの対応（放棄報告時）:**

1. 放棄理由を受け取る
2. 中止モードに切り替える（ステータスを「作業中止」に更新）
3. ユーザーに理由と次のアクションを報告

## Teammate 管理

- `TeamCreate` でチームを作成する
- teammate のライフサイクル（起動・承認・指示・終了）を管理する
- 並列実装: issue-{番号} teammate は最大4つまで同時起動。それ以上は4つ完了後に起動

## ファイル操作の委任

ファイルの読み書きは teammate に委任する（Glob, Grep, Read, Edit, Write は使わない）。
