# 引数なしの起動フロー

以下のサイクルを、作業がなくなるまで繰り返す。

## ステップ1: 情報取得

Notion と GitHub から現状を取得する（この時点では表示しない）:

- Notion: 全ステータスのタスクを優先度順に最大8件取得
- GitHub: 各タスクに紐づくオープンの Issue と PR を取得

## ステップ2: 作業確認待ちの案内

取得結果に「作業確認待ち」のタスクがある場合、最優先として案内する:

> レビュー待ちのタスクがあります。人間によるマージが必要です。

## ステップ3: AskUserQuestion（3タブ同時）

以下を同時に確認する:

タブ1「自動実行」（multiSelect）:

- 仕様の検査（docs-scan）
- 実装の検査（code-scan）
- タスクの計画（task-planner）

タブ2「計画待ち」（multiSelect）:

- 全て
- 取得した計画待ちタスクを優先度順に最大4件

タブ3「作業待ち」（multiSelect）:

- 全て
- 取得した作業待ちタスクを優先度順に最大4件

## ステップ4: タスク一覧の表示

選択後、取得したタスクをステータスごとにグループ化してツリー表示する。タスクが存在しないステータスは表示しない。

```
`作業待ち`

├─ #[**1042**](https://notion.so/1042) 求人検索フィルターの改善
│  ├─ #[**91**](https://github.com/org/repo/issues/91) ページネーション不具合
│  │  └─ #[**112**](https://github.com/org/repo/pull/112) fix: ページネーションのオフセット修正
│  └─ #[**94**](https://github.com/org/repo/issues/94) 文字数カウント不具合
│
└─ #[**1031**](https://notion.so/1031) 企業プロフィール画像のリサイズ対応
   └─ #[**87**](https://github.com/org/repo/issues/87) ログイン後のリダイレクト不具合
      └─ #[**113**](https://github.com/org/repo/pull/113) fix: ログイン後のリダイレクト先を修正

`計画確認待ち`

├─ #[**1038**](https://notion.so/1038) マッチング通知のリアルタイム化
│
└─ #[**1028**](https://notion.so/1028) 応募フォームのバリデーション強化
   └─ #[**96**](https://github.com/org/repo/issues/96) 画像アップロードエラー
```

表示ルール:

- ステータスはインラインコードで表示、直後に空行を入れる
- `#` はリンク外、番号は太字で Notion / GitHub URL にリンク
- Issue はタスクの下に `├─` / `└─` で表示
- PR は Issue の下にさらにインデント
- タスク間に `│` だけの空行を入れて視認性を確保
- ステータスの順序: 未着手 → 計画待ち → 計画中 → 計画確認待ち → 計画中止 → 作業待ち → 作業中_CLAUDE → 作業確認待ち → 作業中止 → 完了 → 中止

## ステップ5: メンバーの起動

TeamCreate でチームを作成し、選択に応じてメンバーを起動する:

- 仕様の検査: `docs-scan` を起動
- 実装の検査: `code-scan` を起動
- タスクの計画: 各 `task-{番号}-plan` を並列起動（copilot-task-planner スキルを指示）
- 作業待ちタスク: 各 `issue-{番号}` を並列起動（copilot-issue スキルを指示）

## ステップ6: 完了後にループ

全メンバーの報告を受け取ったら結果をまとめてユーザーに説明する。その後ステップ1に戻り、作業がなくなるまで繰り返す。
