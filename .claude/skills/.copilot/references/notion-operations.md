# Notion 操作

Notion MCP を使った技術的な操作方法を定義する。

## 基本ルール

Notion MCP で操作する。MCP が使えない場合はエラーを表示して中断する。Web 検索やスクレイピングで代替しない。

Notion のデータは別の Claude や人間が随時変更する可能性がある。操作前に最新状態を確認すること。

## 接続確認

起動時に `notion-search` でデータベースを検索し、データベースのページ ID を取得する。取得したページ ID を `notion-fetch` で取得する。エラーが返った場合はユーザーに伝え、中断する。

注意: 環境変数 `CUSTOM_NOTION_TABLE_TASK_ID` はデータソース ID であり、ページ ID ではない。`notion-fetch` にはページ ID を使う。

## データソース

このデータベースは複数データソースを持つ。fetch 結果の `<data-source>` タグからデータソース URL (`collection://...`) を取得する。タスクのデータソースは環境変数 `CUSTOM_NOTION_TABLE_TASK_ID` に対応する `collection://` の方を使う。

## タスク一覧の取得

`notion-query-database-view` でビューを指定してクエリする。SQL クエリツールは存在しない。

利用可能なビューは `notion-fetch` の結果から取得する。fetch 結果に含まれる `view://` URL の一覧を使う。

copilot のタスク取得には完了以外をフィルタ済みのビュー(「プランニング」等)を使う。

リポジトリで絞り込む場合は、取得結果の `リポジトリ` フィールドでクライアント側フィルタする。

## タスクの作成

`notion-create-pages` でデータソースにページを作成する。`parent` には `data_source_id` を指定する。

作成時に `Claude` タグを必ず付与する。

## ステータス更新時の更新履歴記録

Notion のステータスを更新するたびに、更新履歴データソース（`30b842f9-6181-8026-9a1e-000bd7c11afa`）に記録を追加する。

`notion-create-pages` で `parent.data_source_id` に `30b842f9-6181-8026-9a1e-000bd7c11afa` を指定して作成する。記録する内容:

- タスク名または対象タスクへの参照
- 変更後のステータス
- 変更日時

## 承認タスクの作成

ユーザーの承認が必要になった場合、タスクテーブルに承認タスクをサブタスクとして作成する。

プロパティ:

- 種別: 承認
- タグ: Claude
- 締切: 当日から4日後
- 担当者: `CUSTOM_NOTION_USER_ID`（未設定なら自動取得）
- ステータス: 未着手
- 親タスク: 該当タスク

## ページ本文・コメント・担当者

`notion-task` スキルのガイドラインに従う。
